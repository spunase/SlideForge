# Growth Strategy

Last Updated: 2026-03-04

## 1. Growth Principles

- Growth features cannot weaken privacy defaults.
- Core conversion speed and fidelity remain first priority.
- Sharing is always explicit opt-in.

## 2. Near-Term Growth Initiatives

1. Open-source credibility
- Publish clear roadmap, benchmark data, and support matrix.
- Highlight measurable quality and performance targets.

2. Showcase gallery
- Curated examples generated from non-sensitive sample fixtures.
- No user document upload required for gallery participation.

3. Social proof loops
- One-click share of result summary (timing, slide count, quality score).
- Share payload excludes raw HTML/assets by default.

## 3. Optional Share Modes

### Local-only share (default)
- User shares downloaded PPTX manually.
- No cloud storage involved.

### Hosted preview share (opt-in)
- User explicitly selects slides or rendered images to upload.
- Raw source HTML is excluded unless explicitly enabled with warning.
- Link expiration and delete controls are mandatory.

## 4. Distribution Channels

- Product Hunt launch with benchmark-backed claims.
- Developer channels (GitHub, Hacker News, Reddit) with transparent limits.
- Documentation-led SEO around supported CSS and performance profile.

## 5. Guardrails

- Any growth experiment must pass conversion SLO regression checks.
- Privacy notice must be visible before network-dependent actions.
- Growth metrics cannot collect source document contents.
